# 命名规范文档

## 页面命名规范

### 基本原则
1. **与数据表名对应**：页面路径应尽量与数据库表名对应
2. **使用 kebab-case**：所有页面路径使用短横线命名（kebab-case）
3. **语义清晰**：命名要能清楚表达页面功能

### 数据表与页面对应关系

| 数据表 | 页面路径 | 说明 |
|--------|---------|------|
| `products` | `pages/product/` | 商品列表（单数形式，列表页常用） |
| `products` | `pages/product-detail/` | 商品详情 |
| `categories` | `pages/category-filter/` | 分类筛选 |
| `packages` | `pages/package/` | 套餐列表（单数形式） |
| `packages` | `pages/package-detail/` | 套餐详情 |
| `carts` | `pages/cart/` | 购物车（单数形式） |
| `orders` | `pages/order-list/` | 订单列表 |
| `orders` | `pages/order-detail/` | 订单详情 |
| `orders` | `pages/order/` | 订单提交（创建订单） |
| `orders` | `pages/order-success/` | 订单提交成功（状态页面） |
| `users` | `pages/user-profile/` | 用户资料（完善信息） |
| - | `pages/login/` | 登录页面 |
| - | `pages/mine/` | 我的页面 |
| - | `pages/resource/` | 资料库 |
| - | `pages/upload/` | 文件上传 |

### 命名规则说明

1. **列表页使用单数**：
   - `product`（商品列表，对应 `products` 表）
   - `package`（套餐列表，对应 `packages` 表）
   - `cart`（购物车，对应 `carts` 表）

2. **详情页使用 `-detail` 后缀**：
   - `product-detail`（商品详情）
   - `package-detail`（套餐详情）
   - `order-detail`（订单详情）

3. **列表页使用 `-list` 后缀**（当需要区分时）：
   - `order-list`（订单列表，区别于订单提交页）

4. **操作页使用动词或动作**：
   - `order`（订单提交/创建）
   - `order-success`（订单提交成功）

5. **筛选/过滤页使用 `-filter` 后缀**：
   - `category-filter`（分类筛选）

6. **用户相关使用 `user-` 前缀**：
   - `user-profile`（用户资料，对应 `users` 表）

### 已优化的命名

- ✅ `shopping` → `product`（与 `products` 表对应）
- ✅ `shopping-detail` → `product-detail`
- ✅ `genre-filter` → `category-filter`（与 `categories` 表对应）
- ✅ `login-profile` → `user-profile`（与 `users` 表对应，更准确）
- ✅ `order-succeed` → `order-success`（使用名词形式更规范）

### 注意事项

1. **不要使用旧项目的命名**：旧项目命名不规范，不要参考
2. **保持一致性**：相同类型的页面使用相同的命名模式
3. **优先对应数据表**：如果页面主要操作某个数据表，命名应尽量对应
